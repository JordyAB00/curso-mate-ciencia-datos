La trigonometría es el lenguaje matemático que sustenta los transformers (ChatGPT, Claude, Gemini), el procesamiento de señales y la codificación de datos cíclicos como hora, día o mes.

Funciones trigonométricas en NumPy

Vectorizadas, rápidas y siempre en radianes

import numpy as np # === Conversiones === angulo_deg = 45 angulo_rad = np.radians(angulo_deg) # 0.7854 angulo_deg2 = np.degrees(angulo_rad) # 45.0 # Vectorizar: convertir arrays completos notables = np.array([0, 30, 45, 60, 90, 180, 360]) rads = np.radians(notables) # === Funciones directas === theta = np.linspace(0, 2*np.pi, 360) seno = np.sin(theta) # 360 valores coseno = np.cos(theta) # simultáneamente tangente = np.tan(theta) # === Funciones inversas === vals = np.array([-1, -0.5, 0, 0.5, 1.0]) np.arcsin(vals) # [-π/2, ..., π/2] np.arccos(vals) # [π, ..., 0] np.arctan(vals) # (-π/2, π/2) # arctan2: resuelve el cuadrante correcto y = np.array([1, -1, -1, 1]) x = np.array([1, 1, -1, -1]) np.arctan2(y, x) # [π/4, -π/4, -3π/4, 3π/4]
Función NumPy Dominio → Rango
Seno np.sin(x) R → [−1, 1]
Coseno np.cos(x) R → [−1, 1]
Tangente np.tan(x) R\{π/2+nπ} → R
Arcoseno np.arcsin(x) [−1,1] → [−π/2, π/2]
Arcocoseno np.arccos(x) [−1,1] → [0, π]
Arcotangente np.arctan(x) R → (−π/2, π/2)

Aliases en NumPy 2.x

Desde NumPy 2.0 existen aliases cortos: np.asin, np.acos, np.atan. Las versiones np.arcsin, np.arccos, etc. siguen vigentes y no están deprecadas.

Regla de oro: NumPy siempre trabaja en radianes. np.sin(90) NO da 1 porque 90 se interpreta como radianes. Siempre convertir: np.sin(np.radians(90)).

El truco de np.nan (de nuevo)

Igual que con funciones racionales, reemplazamos valores extremos de tan(x) con np.nan para evitar líneas espurias. Matplotlib no dibuja líneas donde hay nan.

Relación fundamental

cos(x) = sin(x + π/2). El coseno es el seno desplazado π/2 a la izquierda. Esta relación de fase aparece en la codificación posicional de los transformers.

Identidades trigonométricas en SymPy

trigsimp(), expand_trig() y verificación automática

import sympy as sp x, y = sp.symbols('x y') # === Verificar identidad pitagórica === expr = sp.sin(x)**2 + sp.cos(x)**2 print(sp.trigsimp(expr)) # → 1 # === Expandir identidades de suma === print(sp.expand_trig(sp.sin(x + y))) # → sin(x)cos(y) + sin(y)cos(x) print(sp.expand_trig(sp.cos(x + y))) # → cos(x)cos(y) - sin(x)sin(y) # === Ángulo doble === print(sp.expand_trig(sp.sin(2*x))) # → 2*sin(x)*cos(x) print(sp.expand_trig(sp.cos(2*x))) # → 2*cos(x)**2 - 1 # === Simplificación inversa === expr2 = sp.sin(x)*sp.cos(y) + sp.cos(x)*sp.sin(y) print(sp.trigsimp(expr2)) # → sin(x + y) # === Simplificación avanzada === expr3 = sp.sin(x)*sp.tan(x)/sp.sec(x) print(sp.trigsimp(expr3)) # → sin(x)**2
Función SymPy Qué hace
sp.trigsimp(expr) Simplifica: reduce a forma más simple
sp.expand_trig(expr) Expande: sin(A+B) → componentes
expr.rewrite(sp.cos) Reescribe todo en términos de cos
sp.trigsimp(inverse=True) Cancela asin(sin(x)) → x

trigsimp vs expand_trig

trigsimp() comprime: convierte sumas de productos en funciones de sumas. expand_trig() expande: descompone sin(A+B) en sus componentes. Son operaciones inversas.

SymPy verifica identidades automáticamente. Si trigsimp(expresión) da un resultado simple (como 1), acabamos de demostrar una identidad.

Ecuaciones trigonométricas con SymPy

solveset() para soluciones generales y en intervalos

from sympy import * x = symbols('x') # === sin(x) = 1/2: solución general === sol = solveset(sin(x) - Rational(1,2), x, domain=S.Reals) print(sol) # Union(ImageSet(n, 2nπ+π/6), # ImageSet(n, 2nπ+5π/6)) # === sin(x) = 1/2 en [0, 2π) === sol_int = solveset(sin(x) - Rational(1,2), x, domain=Interval(0, 2*pi, False, True)) print(sol_int) # {π/6, 5π/6} # === 2sin²(x) - 1 = 0 en [0, 2π) === sol2 = solveset(2*sin(x)**2 - 1, x, domain=Interval(0, 2*pi, False, True)) print(sol2) # {π/4, 3π/4, 5π/4, 7π/4} # === cos(2x) = cos(x) en [0, 2π) === sol3 = solveset(cos(2*x) - cos(x), x, domain=Interval(0, 2*pi, False, True)) print(sol3)
sin(x) = 1/2 en [0, 2π): {π/6, 5π/6} 2sin²(x) - 1 = 0: {π/4, 3π/4, 5π/4, 7π/4}

Interval(0, 2*pi, False, True)

El tercer argumento indica si el extremo izquierdo es abierto (False = cerrado). El cuarto, si el derecho es abierto (True = abierto). Así obtenemos [0, 2π).

Rational(1, 2) vs 0.5

Usar Rational(1, 2) o S(1)/2 para obtener soluciones exactas. Con 0.5 (flotante), SymPy puede dar aproximaciones numéricas en vez de fracciones de π.

Aplicación: si modelamos generación solar como P(t) = A·sin(2πt/24 − π/2), resolver P(t) = umbral nos dice las horas del día en que la generación supera ese nivel.

Codificación cíclica de variables temporales

El problema: hora 23 está cerca de hora 0, pero numéricamente parecen lejanas

import numpy as np import pandas as pd def codificar_ciclico(df, columna, periodo): """Codifica variable cíclica con sin/cos.""" df[f'{columna}_sin'] = np.sin( 2 * np.pi * df[columna] / periodo) df[f'{columna}_cos'] = np.cos( 2 * np.pi * df[columna] / periodo) return df # Ejemplo: codificar hora del día df = pd.DataFrame({'hora': range(24)}) df = codificar_ciclico(df, 'hora', 24) # Verificar: hora 23 y hora 0 están CERCA h23 = np.array([df.loc[23, 'hora_sin'], df.loc[23, 'hora_cos']]) h0 = np.array([df.loc[0, 'hora_sin'], df.loc[0, 'hora_cos']]) dist = np.linalg.norm(h23 - h0) print(f"Distancia 23→0: {dist:.3f}") # ≈ 0.264 (cerca, como debe ser) # Comparar: distancia 23→12 h12 = np.array([df.loc[12, 'hora_sin'], df.loc[12, 'hora_cos']]) dist2 = np.linalg.norm(h23 - h12) print(f"Distancia 23→12: {dist2:.3f}") # ≈ 1.802 (lejos, correcto)

El problema

Si usamos la hora como número crudo (0, 1, ..., 23), un modelo de ML pensaría que hora 23 y hora 0 están a distancia 23. Pero en realidad están a 1 hora de diferencia.

La solución: mapear al círculo unitario

xsin = sin(2π · x / T), xcos = cos(2π · x / T), donde T es el período. Cada valor se convierte en un punto único del círculo unitario (sin, cos).

¿Por qué necesitamos AMBAS componentes?

Solo con seno: sin(6h) = sin(18h) para T=24. No podríamos distinguir las 6am de las 6pm. El coseno rompe esa ambigüedad: cos(6h) ≠ cos(18h).

Períodos comunes: hora → T=24, día de semana → T=7, mes → T=12, día del año → T=365.25. Esto aplica directamente en sus proyectos.

import numpy as np import matplotlib.pyplot as plt # === Radar chart: matriz energética CR === fuentes = ['Hidro', 'Eólica', 'Geo', 'Solar', 'Biomasa', 'Térmica'] pcts = [67.5, 15.3, 13.8, 1.8, 0.9, 0.7] # Ángulos equidistantes en el círculo n = len(fuentes) angulos = np.linspace(0, 2*np.pi, n, endpoint=False).tolist() # Cerrar el polígono vals = pcts + pcts[:1] angulos += angulos[:1] fig, ax = plt.subplots(figsize=(7,7), subplot_kw=dict(polar=True)) ax.plot(angulos, vals, 'o-', lw=2, color='#006847') ax.fill(angulos, vals, alpha=0.25, color='#006847') ax.set_thetagrids( np.degrees(angulos[:-1]), fuentes) ax.set_theta_offset(np.pi/2) ax.set_theta_direction(-1) ax.set_title('Matriz energética de CR') plt.show()

Ángulos equidistantes

Cada variable se ubica a θi = 2πi/n. Con 6 fuentes: 0°, 60°, 120°, 180°, 240°, 300°. Esto es exactamente el círculo unitario con n puntos.

Proyección polar en Matplotlib

subplot_kw=dict(polar=True) activa coordenadas polares. set_theta_offset(π/2) pone el 0° arriba. set_theta_direction(-1) gira en sentido horario.

Generación solar: sinusoide casi perfecta

Los datos del CENCE muestran que la generación solar sigue un patrón P(t) ≈ A·sin(2πt/24 − π/2): cero en la noche, máximo al mediodía, y sigue la forma de una sinusoide positiva. La generación eólica tiene un patrón complementario, desfasado aproximadamente π radianes respecto a la solar.

Estos patrones cíclicos son ideales para aplicar codificación sin/cos. Si su proyecto usa datos temporales, esta técnica mejora significativamente los modelos.

Codificación posicional en transformers

Cómo ChatGPT, Claude y Gemini saben la posición de cada palabra

import numpy as np import matplotlib.pyplot as plt def codificacion_posicional(max_len, d_model): """Codificación posicional sinusoidal.""" pe = np.zeros((max_len, d_model)) pos = np.arange(max_len)[:, np.newaxis] div = np.exp(np.arange(0, d_model, 2) * -(np.log(10000) / d_model)) pe[:, 0::2] = np.sin(pos * div) # pares pe[:, 1::2] = np.cos(pos * div) # impares return pe # Generar y visualizar pe = codificacion_posicional(50, 64) fig, ax = plt.subplots(figsize=(12, 5), layout='constrained') im = ax.imshow(pe, aspect='auto', cmap='RdYlBu') ax.set_xlabel('Dimensión del modelo') ax.set_ylabel('Posición de la palabra') ax.set_title('Codificación posicional sinusoidal') fig.colorbar(im, ax=ax) plt.show()

La idea central

Los transformers procesan todas las palabras en paralelo (no secuencialmente). Necesitan un mecanismo para saber dónde está cada palabra. La solución: sumar un vector de senos y cosenos único para cada posición.

Las fórmulas

PE(pos, 2i) = sin(pos / 100002i/d)
PE(pos, 2i+1) = cos(pos / 100002i/d)
Cada dimensión usa una frecuencia diferente: las primeras cambian rápido, las últimas lento.

¿Por qué sin y cos?

Gracias a las identidades de suma: PE(pos+k) es una rotación de PE(pos). Esto permite al modelo aprender relaciones posicionales relativas, no solo absolutas.

Esta es la aplicación directa de las identidades de suma que vimos en el iPad: sin(A+B) = sinA·cosB + cosA·sinB es lo que permite que la posición (pos+k) sea una transformación lineal de la posición (pos).

Funciones hiperbólicas y activaciones neuronales

tanh y GELU: la trigonometría dentro de las redes neuronales

import numpy as np x = np.linspace(-5, 5, 1000) # === Funciones hiperbólicas === sinh = np.sinh(x) # (e^x - e^-x)/2 cosh = np.cosh(x) # (e^x + e^-x)/2 tanh = np.tanh(x) # sinh/cosh # === tanh como activación === # Rango: (-1, 1), centrada en 0 # Usada en RNNs, LSTMs activacion_tanh = np.tanh(x) # === GELU (aprox. tanh) === # Activación estándar en BERT, GPT def gelu(x): return 0.5 * x * (1 + np.tanh( np.sqrt(2/np.pi) * (x + 0.044715 * x**3))) activacion_gelu = gelu(x) # === Identidad hiperbólica === check = np.cosh(x)**2 - np.sinh(x)**2 print(np.allclose(check, 1)) # True # Análogo a sin² + cos² = 1 # pero con signo opuesto: cosh² - sinh² = 1
tanh

Rango: (−1, 1)

Centrada en 0, suave. Usada en RNNs y como gate en LSTMs. tanh(x) = (ex−e−x)/(ex+e−x).

GELU

Activación de GPT

GELU(x) ≈ 0.5x(1+tanh(√(2/π)(x+0.04x³))). Contiene tanh y la constante √(2/π).

Sigmoid

σ(x) = (1+tanh(x/2))/2

La sigmoide se puede escribir en términos de tanh. Rango: (0, 1). Usada en clasificación binaria.

Conexión con trig circular

La relación entre funciones circulares e hiperbólicas: tanh(x) = −i·tan(ix). Las hiperbólicas son las circulares evaluadas en argumentos imaginarios. Misma estructura, diferente geometría.

Cada vez que un LLM genera una palabra, la aproximación tanh de GELU se calcula billones de veces. La tangente hiperbólica del precálculo es infraestructura del deep learning moderno.

Similaridad coseno y análisis de Fourier

La métrica trigonométrica más usada en ML + detección de periodicidad

import numpy as np # === Similaridad coseno === def sim_coseno(a, b): """cos(θ) entre vectores a y b.""" return np.dot(a, b) / ( np.linalg.norm(a) * np.linalg.norm(b)) # Perfiles turísticos de provincias CR # [playa, montaña, cultura, aventura, gastro] san_jose = np.array([0.2,0.3,0.9,0.4,0.8]) guanacaste = np.array([0.95,0.2,0.3,0.7,0.5]) cartago = np.array([0.1,0.9,0.7,0.5,0.4]) print(f"SJ-Cartago: {sim_coseno(san_jose, cartago):.3f}") print(f"SJ-Guanacaste: {sim_coseno(san_jose, guanacaste):.3f}") # === FFT: detectar periodicidad === from scipy.fft import rfft, rfftfreq # Simular turismo mensual (10 años) meses = np.arange(120) turismo = (200000 + 1000*meses + 50000*np.sin(2*np.pi*meses/12) + np.random.normal(0, 5000, 120)) fft_vals = rfft(turismo - turismo.mean()) freqs = rfftfreq(len(turismo)) magnitud = np.abs(fft_vals) # Pico en freq = 1/12 → período de 12 meses

Similaridad coseno

cos(θ) = (a · b) / (||a|| · ||b||). Mide el ángulo entre vectores, no su magnitud. Rango: [−1, 1]. Es la métrica estándar en NLP (comparar embeddings de palabras) y en búsqueda semántica.

FFT (transformada rápida de Fourier)

Descompone cualquier señal en sus frecuencias componentes. Si una serie temporal tiene estacionalidad, la FFT la detecta como un pico en la frecuencia correspondiente (1/12 para ciclos anuales en datos mensuales).

Idea de Fourier en una frase

Cualquier señal periódica se puede escribir como suma de senos y cosenos de diferentes frecuencias. Las identidades trigonométricas de suma son el fundamento algebraico de esta descomposición.

Para sus proyectos: si sospechan que sus datos tienen estacionalidad, usen scipy.fft.rfft() para confirmarla. Los picos en el espectro les dicen exactamente cuál es el período.
