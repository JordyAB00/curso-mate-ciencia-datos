Las funciones racionales modelan saturación y rendimientos decrecientes: retorno publicitario, adopción tecnológica, curvas de rendimiento. SymPy para analizar, Matplotlib para graficar correctamente.

Definir y simplificar funciones racionales

SymPy 1.14: cancel(), factor(), fraction() y apart()

import sympy as sp x = sp.Symbol('x') # Definir una función racional P = x**3 - 2*x**2 - 5*x + 6 Q = x**2 - 4 f = P / Q # Factorizar numerador y denominador print("P(x):", sp.factor(P)) # => (x - 3)(x - 1)(x + 2) print("Q(x):", sp.factor(Q)) # => (x - 2)(x + 2) # Simplificar (cancela factores comunes) f_simple = sp.cancel(f) print("Simplificada:", f_simple) # => (x**2 - 4*x + 3)/(x - 2) # Extraer numerador y denominador num, den = sp.fraction(f_simple) print("Num:", num, "Den:", den) # Fracciones parciales print("Parciales:", sp.apart(f, x)) # Clasificar: propia vs impropia grado_P = sp.degree(P, x) # 3 grado_Q = sp.degree(Q, x) # 2 tipo = "propia" if grado_P < grado_Q else "impropia" print(f"Tipo: {tipo}") # impropia
Función SymPy Qué hace
sp.factor(expr) Factoriza un polinomio
sp.cancel(f) Cancela factores comunes P/Q
sp.fraction(f) Separa en (numerador, denominador)
sp.apart(f, x) Descompone en fracciones parciales
sp.degree(P, x) Grado del polinomio
sp.div(P, Q, x) División polinomial (cociente, residuo)

Hueco vs. asíntota

cancel() elimina factores comunes. Si (x+2) estaba en P y Q y desapareció al simplificar, hay un hueco en x = −2. Si un factor queda en el denominador simplificado, es asíntota vertical.

Protocolo "Dominio Primero": siempre factorizar y simplificar con cancel() ANTES de cualquier otro análisis. Esto clasifica automáticamente huecos vs. asíntotas.

Asíntotas verticales con SymPy

solveset() + limit() para encontrar y verificar singularidades

import sympy as sp x = sp.Symbol('x') f = (2*x**2 + 3*x - 5) / (x**2 - x - 6) # Paso 1: Simplificar f_simple = sp.cancel(f) num, den = sp.fraction(f_simple) print("Simplificada:", f_simple) # Paso 2: Ceros del denominador simplificado asintotas_v = sp.solveset(den, x, sp.S.Reals) print("Asíntotas verticales:", asintotas_v) # => {-2, 3} # Paso 3: Verificar con límites laterales for a in asintotas_v: lim_der = sp.limit(f, x, a, '+') lim_izq = sp.limit(f, x, a, '-') print(f"x -> {a}+: {lim_der}") print(f"x -> {a}-: {lim_izq}") # También: usar singularities() from sympy.calculus.singularities import singularities puntos = singularities(f, x) print("Singularidades:", puntos)
Simplificada: (2*x**2 + 3*x - 5)/(x**2 - x - 6) Asíntotas verticales: {-2, 3} x -> -2+: -oo x -> -2-: +oo x -> 3+: +oo x -> 3-: -oo Singularidades: {-2, 3}

Límites laterales

sp.limit(f, x, a, '+') calcula el límite por la derecha. '-' por la izquierda. Si dan signos opuestos (como en x = 3), el denominador tiene un cero de multiplicidad impar.

Identificar huecos automáticamente

Comparar ceros del denominador original vs. el simplificado. Los que desaparecen al simplificar son huecos. Los que permanecen son asíntotas verticales.

La combinación solveset() + limit() reproduce exactamente lo que hicimos a mano en el iPad, pero con verificación automática.

Asíntotas horizontales y oblicuas

limit() hacia infinito y división polinomial con div()

import sympy as sp x = sp.Symbol('x') # === CASO 1: grado P < grado Q => AH: y = 0 === f1 = 3 / (x**2 + 1) print("f1 AH:", sp.limit(f1, x, sp.oo)) # 0 # === CASO 2: grado P = grado Q => AH: y = an/bm === f2 = (2*x**2 + 3*x - 5) / (x**2 - x - 6) print("f2 AH:", sp.limit(f2, x, sp.oo)) # 2 # === CASO 3: grado P > grado Q => No hay AH === f3 = (x**2 + 3*x + 1) / (x - 1) print("f3 lím:", sp.limit(f3, x, sp.oo)) # oo # Asíntota oblicua: dividir P entre Q P3 = x**2 + 3*x + 1 Q3 = x - 1 cociente, residuo = sp.div(P3, Q3, x) print(f"Obl: y = {cociente}") # x + 4 # Verificar: f3(x) - oblicua -> 0 print(sp.limit(f3 - cociente, x, sp.oo)) # 0
Relación de grados Asíntota Código
deg(P) < deg(Q) AH: y = 0 limit(f, x, oo) => 0
deg(P) = deg(Q) AH: y = an/bm limit(f, x, oo) => a/b
deg(P) = deg(Q) + 1 Oblicua: y = mx + b sp.div(P, Q, x)
deg(P) > deg(Q) + 1 No hay limit => ±oo

Hecho importante

Una función racional puede cruzar su asíntota horizontal. La AH describe el comportamiento cuando x → ±∞, no es una barrera. Ejemplo: f(x) = x/(x² + 1) cruza y = 0 en x = 0.

En ciencia de datos, la asíntota horizontal es el valor de saturación: el máximo retorno publicitario, la capacidad máxima de turistas, el límite de precisión de un modelo.

Dominio y clasificación de discontinuidades

continuous_domain() y análisis automático de huecos vs. asíntotas

import sympy as sp from sympy.calculus.util import continuous_domain x = sp.Symbol('x') # f(x) = (x² - x - 6)/(x² - 9) # = (x-3)(x+2) / (x-3)(x+3) f = (x**2 - x - 6) / (x**2 - 9) # Dominio continuo dom = continuous_domain(f, x, sp.S.Reals) print("Dominio:", dom) # => (-oo, -3) U (-3, 3) U (3, oo) # Clasificar cada exclusión num_orig = sp.numer(f) den_orig = sp.denom(f) ceros_den = sp.solveset(den_orig, x, sp.S.Reals) print("Excluidos:", ceros_den) # {-3, 3} # Simplificar y comparar f_simple = sp.cancel(f) _, den_simple = sp.fraction(f_simple) for c in ceros_den: if den_simple.subs(x, c) == 0: print(f"x={c}: Asíntota vertical") else: y_hueco = f_simple.subs(x, c) print(f"x={c}: Hueco en ({c}, {y_hueco})")
Dominio: (-oo, -3) U (-3, 3) U (3, oo) Excluidos: {-3, 3} x=3: Hueco en (3, 5/6) x=-3: Asíntota vertical
Hueco

x = 3

Factor (x−3) se cancela. La función simplificada vale 5/6 ahí. Punto: (3, 5/6).

Asíntota V.

x = −3

Factor (x+3) permanece en el denominador. La función tiende a ±∞.

En datos

División por cero

Costo promedio AC(q) = C(q)/q tiene AV en q = 0: producir cero unidades tiene costo infinito.

El algoritmo es mecánico: factorizar, simplificar con cancel(), comparar denominadores antes y después. Lo que desaparece es hueco, lo que queda es asíntota.

El truco de np.nan y funciones inversas con SymPy

Graficar funciones racionales en Matplotlib

El problema de las líneas espurias y la solución con np.nan

import numpy as np import matplotlib.pyplot as plt # f(x) = (2x+1) / (x² - x - 6) x = np.linspace(-8, 8, 2000) num = 2*x + 1 den = x**2 - x - 6 # = (x-3)(x+2) with np.errstate(divide='ignore', invalid='ignore'): y = num / den # CLAVE: reemplazar valores extremos con NaN y[np.abs(y) > 50] = np.nan # También: NaN cerca de cada asíntota vertical for av in [3, -2]: mask = np.abs(x - av) < 0.05 y[mask] = np.nan # Graficar fig, ax = plt.subplots(figsize=(10, 7), layout='constrained') ax.plot(x, y, 'b-', lw=2, label='f(x)') # Asíntotas verticales (rojo punteado) ax.axvline(x=3, color='red', ls='--', alpha=0.7, label='AV: x=3') ax.axvline(x=-2, color='red', ls='--', alpha=0.7, label='AV: x=-2') # Asíntota horizontal (verde punteado) ax.axhline(y=0, color='green', ls='--', alpha=0.7, label='AH: y=0') ax.set_xlim(-8, 8); ax.set_ylim(-15, 15) ax.legend(); ax.grid(True, alpha=0.3) ax.set_title('f(x) = (2x+1)/(x²-x-6)') plt.show()

El problema

Matplotlib conecta puntos consecutivos con líneas rectas. Cerca de una asíntota vertical, esto genera una línea vertical espuria que no existe en la función.

La solución: np.nan

Reemplazar con np.nan los valores cercanos a las asíntotas y los valores extremos. nan rompe la línea del gráfico, eliminando el artefacto visual.

Código de colores estándar

  • Curva de la función: azul sólido (lw=2)
  • Asíntota vertical: rojo punteado ('--')
  • Asíntota horizontal: verde punteado ('--')
  • Asíntota oblicua: naranja punteado ('--')
  • Hueco: círculo blanco con borde azul

Usar siempre errstate(divide='ignore') para evitar warnings de división por cero. NumPy genera inf y nan que luego limpiamos manualmente.

Funciones inversas con SymPy

solve() para despejar, simplify() para verificar composición

import sympy as sp x, y = sp.symbols('x y') # f(x) = (2x + 3) / (x - 1) f = (2*x + 3) / (x - 1) # Paso 1: Resolver y = f(x) para x inversa_sol = sp.solve(sp.Eq(y, f), x) print("f_inv(y) =", inversa_sol) # => [(y + 3)/(y - 2)] # Paso 2: Reescribir con x f_inv = inversa_sol[0].subs(y, x) print("f_inv(x) =", f_inv) # => (x + 3)/(x - 2) # Paso 3: Verificar composición comp1 = sp.simplify(f.subs(x, f_inv)) print("f(f_inv(x)) =", comp1) # x comp2 = sp.simplify(f_inv.subs(x, f)) print("f_inv(f(x)) =", comp2) # x # Dominios y rangos print("Dom(f): R \\ {1}") print("Ran(f): R \\ {2}") print("Dom(f_inv): R \\ {2} = Ran(f)") print("Ran(f_inv): R \\ {1} = Dom(f)")
f_inv(y) = [(y + 3)/(y - 2)] f_inv(x) = (x + 3)/(x - 2) f(f_inv(x)) = x f_inv(f(x)) = x

Método: resolver para x

En vez de "intercambiar x e y" (que genera confusión), despejamos x de la ecuación y = f(x). El resultado es la inversa. Luego renombramos y → x si se desea.

Propiedad clave

Dom(f−1) = Ran(f) y Ran(f−1) = Dom(f). Las asíntotas de f y f−1 se intercambian: la AV de f se convierte en AH de f−1 y viceversa.

En ciencia de datos: scaler.inverse_transform() deshace StandardScaler. En GLMs, la función de enlace inversa mapea predicciones lineales a la escala original. Son inversas funcionales.

Aplicación CR: modelo de saturación con datos reales

Ajustar una función racional a datos de turismo (ICT) con curve_fit

import numpy as np from scipy.optimize import curve_fit import matplotlib.pyplot as plt # Modelo racional de saturación # f(x) = V_max * x / (K_m + x) def saturacion(x, V_max, K_m): return (V_max * x) / (K_m + x) # Datos ICT: turistas a CR (miles) por década anios = np.array([1970,1980,1990,2000,2005, 2010,2015,2019,2023]) turistas = np.array([160,260,435,1088,1679, 2100,2660,3139,2900]) # x = años desde 1960 x_data = anios - 1960 # Ajustar modelo racional popt, pcov = curve_fit(saturacion, x_data, turistas, p0=[5000, 30]) V_max, K_m = popt print(f"Asíntota horizontal: {V_max:.0f} mil turistas") print(f"K_m: {K_m:.1f} años (media saturación)") # Graficar t_plot = np.linspace(5, 80, 500) fig, ax = plt.subplots(figsize=(10,6), layout='constrained') ax.scatter(x_data, turistas, c='#2d2d2d', s=80, zorder=5, label='Datos ICT') ax.plot(t_plot, saturacion(t_plot, *popt), color='#FF8C00', lw=2, label='Modelo racional') ax.axhline(y=V_max, color='green', ls='--', alpha=0.7, label=f'AH: {V_max:.0f} mil') ax.set_xlabel('Años desde 1960') ax.set_ylabel('Turistas (miles)') ax.set_title('Turismo en Costa Rica: modelo de saturación') ax.legend(); ax.grid(True, alpha=0.3) plt.show()

El modelo Vmaxx / (Km + x)

Es una función racional propia (grado 1/grado 1). La asíntota horizontal es y = Vmax: el máximo de turistas que la infraestructura de CR puede absorber. Km indica cuándo se alcanza la mitad.

curve_fit de SciPy

  • Ajusta cualquier función definida por el usuario a datos
  • Retorna parámetros óptimos (popt) y covarianza (pcov)
  • p0 = estimación inicial (ayuda a la convergencia)
  • Funciona con funciones racionales, exponenciales, cualquiera

Otros fenómenos modelables

  • Energía renovable de CENCE: se acerca al 100%
  • Curva de rendimiento soberana del BCCR
  • Retorno publicitario: saturación del gasto en marketing
  • Precisión de ML vs. tamaño del dataset
